# Mini Agent Orchestrator v0.3 - BYOK LLM Execution

## Goal

v0.3 adds optional direct LLM execution for the existing fixed workflow while keeping the CLI local, explicit, and testable.

LLM mode is BYOK: bring your own key. Users provide provider credentials from their own local environment or local configuration. Mini Agent Orchestrator does not provide hosted inference, does not proxy requests through a project-owned backend, and does not store provider keys on a server.

## Relationship to v0.2

v0.2 introduced prompt-only mode with `--include-prompts`. In that mode, the CLI writes copyable prompt files and never calls an LLM API.

v0.3 keeps that prompt-only behavior available. LLM execution is a separate opt-in mode. The default command still runs the deterministic local workflow unless the user explicitly enables LLM execution.

## Workflow relationship

The workflow remains fixed:

```text
idea -> spec -> tasks -> review checklist
```

In v0.3 LLM mode, the CLI should execute the same stages through the configured provider and write the generated Markdown artifacts to the same project output directory:

```text
artifacts/<project-name>/
  idea.md
  spec.md
  tasks.md
  review-checklist.md
  metadata.json
```

If `--include-prompts` is used with LLM mode, prompt files may still be written as companion artifacts under `prompts/`. Prompt generation does not replace the final generated artifacts.

## BYOK mode

BYOK mode means:

- LLM execution must be explicitly enabled, for example with a future `--llm` flag.
- The user supplies their own provider API key locally.
- API keys should come from environment variables or local config, not from command-line API key flags.
- Provider credentials must never be committed, written into generated artifacts, printed in logs, or included in error messages.
- Running without LLM mode must not require provider credentials.

Suggested environment variables for v0.3:

```text
MAO_LLM_PROVIDER=openai-compatible
MAO_LLM_BASE_URL=https://api.openai.com/v1
MAO_LLM_MODEL=gpt-4.1-mini
MAO_LLM_API_KEY=...
```

## Provider strategy

v0.3 should support one concrete provider path first: an OpenAI-compatible HTTP provider.

The provider strategy is intentionally narrow:

- Support a configurable OpenAI-compatible base URL, model, and API key.
- Use non-streaming request/response behavior.
- Return generated Markdown content for each fixed workflow stage.
- Keep provider-specific aliases and marketplace-style provider discovery out of v0.3.
- Avoid provider SDK dependencies unless the standard library HTTP client is not sufficient.

## Missing configuration and failures

LLM mode should fail before making provider calls when required configuration is missing or invalid.

Expected failure behavior:

- If LLM mode is enabled and `MAO_LLM_API_KEY` is missing, return a clear error that an API key is required.
- If provider, base URL, or model configuration is missing, return a clear validation error.
- Error messages must identify the missing setting without revealing any secret value.
- Default deterministic mode and v0.2 prompt-only mode should continue to work without any LLM configuration.
- Provider call failures, non-2xx responses, timeouts, and context cancellation should stop the LLM workflow and return actionable errors without writing secrets.

## Non-goals

v0.3 does not include:

- user registration
- login or OAuth flows
- hosted backend
- hosted free LLM access
- anonymous users using a project-owned provider key
- billing
- account management
- server-side quota management
- server-side provider key storage
- provider marketplace UX
- streaming
- tool calls
- JSON mode or structured output contracts
- configurable workflows
- multi-agent orchestration

## Success criteria

- v0.3 scope is documented.
- BYOK mode is defined clearly.
- The supported v0.3 provider strategy is documented.
- The docs explain how LLM execution relates to `idea -> spec -> tasks -> review checklist`.
- Failure behavior for missing credentials and provider config is documented.
- Non-goals are explicit, including registration and hosted free LLM access.
- The difference between v0.2 prompt-only mode and v0.3 LLM execution is clear.
